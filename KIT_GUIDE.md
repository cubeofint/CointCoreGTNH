# Руководство по системе наборов (kit)

Это руководство описывает текущее поведение системы наборов в `CointCoreGTNH` для игроков и администраторов.

## 1) Быстрый обзор

- Базовая команда: `/kit`
- Подкоманды: `claim`, `list`, `create`, `delete`, `add`, `set`
- **Кулдаунов на наборах нет** — ограничения только через лимит использований (`maxClaims`) и привилегии.
- Доступ к набору определяется так:
  - **безлимитные наборы** (`maxClaims == -1`): глобально `cointcore.kit.all` (удобно админам) или точечно `cointcore.kit.<имя>`;
  - **лимитные наборы** (`maxClaims > 0`): в основном через **остаток** выданный `/kit add`/`set` (покупка), либо точечно `cointcore.kit.<имя>` для особых случаев.

## 2) Для игроков

### Просмотр доступных наборов

- Команда: `/kit list`
- Показывает **только** наборы, которые игрок может получить **сейчас**:
  - **безлимитные** (`maxClaims == -1`) — при доступе по рангу **или** при ненулевом **бонусном** остатке от `/kit add`/`set`; если бонус есть, в скобках показывается остаток, например `(7)`;
  - **лимитные** — пока не исчерпан лимит; в строке **использовано/всего**, например `(4/10)`.

Если доступных наборов нет, выводится:
- `Нет доступных наборов`

### Получение набора

- Команда: `/kit claim <name>`
- Перед выдачей проверяется, что весь набор полностью помещается в инвентарь.

Если место есть, далее логика такая:
- если у игрока есть персональные оставшиеся использования (лимитный набор и `claims < maxClaims`), набор выдаётся и счётчик увеличивается;
- иначе для **безлимитного** набора нужен `cointcore.kit.all` или `cointcore.kit.<name>`;
- для **лимитного** без остатка — только `cointcore.kit.<name>` (глобальный `all` на такие наборы не распространяется).

### Типовые сообщения об ошибках

- `Набор не найден: <name>` — такого набора нет.
- `Инвентарь полон, набор не получен` — набор не влезает целиком.
- `У вас нет доступа к набору: <name>` — нет ни привилегии, ни остатка по `maxClaims`.

## 3) Для администраторов

### Права (permissions)

- `cointcore.command.kit.edit` — создание и удаление наборов.
- `cointcore.command.kit.reset` — управление остатком использований (`/kit add`, `/kit set`).
- `cointcore.kit.all` — доступ ко **всем безлимитным** наборам (`maxClaims == -1`); мод регистрирует ноду; выдайте администраторам в ServerUtilities.
- `cointcore.kit.<name>` — доступ к конкретному набору (любой тип лимита); прописывается в рангах **вручную** при необходимости узкой выдачи.

### Создание набора

Команда:
- `/kit create <name> [maxClaims]`

Требования:
- имя: только `a-z`, `0-9`, `_`, `-` (до 32 символов);
- источник предметов:
  - контейнер, в который смотрит админ;
  - если контейнер не найден, берется предмет из руки.

Параметр `maxClaims` (**по умолчанию безлимит**):
- не указывать второй аргумент или явно `-1` — безлимитное число получений (доступ через `cointcore.kit.all` и/или `cointcore.kit.<name>`);
- положительное число — лимит получений на игрока (магазин: выдача остатка через `/kit set`/`add`).

После создания:
- набор сохраняется в `cointcore/kits.dat`;
- **permission `cointcore.kit.<name>` мод не регистрирует** — при необходимости добавьте строку доступа в конфиге ServerUtilities / рангов вручную (или полагайтесь только на лимит `maxClaims` и `/kit set`/`add`).

Примеры:
- `/kit create starter`
- `/kit create starter -1`
- `/kit create weekly 5`

### Удаление набора

- `/kit delete <name>`

Удаляет набор из хранилища наборов.

### Управление остатком использований (`maxClaims`) игрока

- `/kit add <name> <player> <amount>`
- `/kit set <name> <player> <amount>`

Где:
- `add` — добавить к текущему остатку;
- `set` — установить остаток в точное значение;
- `amount` — целое число `>= 0`.

Поведение:
- **лимитный набор** (`maxClaims > 0`): считается остаток в рамках лимита набора; сверху обрезается по `maxClaims`;
- **безлимитный набор** (`maxClaims == -1`): ведётся отдельный счётчик **бонусных** получений (магазин): `/kit add fusion Player 10` даёт 10 раз забрать набор **без** ранга, пока бонус не израсходуется; после этого снова нужны `cointcore.kit.all` / `cointcore.kit.<имя>`.

Работает для онлайн и оффлайн игроков.

## 4) Редактирование наборов (как сейчас)

Отдельной команды `/kit edit` в текущей версии нет.

Практический способ редактирования:
1. удалить старый набор: `/kit delete <name>`;
2. создать заново: `/kit create <name> [maxClaims]`.

Если нужно изменить персональный остаток использований:
- использовать `/kit add ...` или `/kit set ...`.

## 5) Как работает `maxClaims`

`maxClaims` хранится в определении набора и задает персональный лимит получений.

- если у игрока есть оставшиеся использования (`claims < maxClaims`), он может получить лимитный набор **без** ранговой привилегии;
- если оставшихся нет: для **лимитного** набора нужна `cointcore.kit.<name>` (или снова выдать остаток через `add`/`set`);
- при `maxClaims == -1` безлимит по определению набора — доступ по рангу через `cointcore.kit.all` или `cointcore.kit.<name>`; дополнительно можно выдать **бонусные** получения через `/kit add`/`set` (см. выше).

## 6) Практические сценарии

### Администраторы (все безлимитные наборы)

1. Выдать в ServerUtilities привилегию `cointcore.kit.all` нужной группе (например, staff).
2. Любой набор с `maxClaims == -1` будет доступен без перечисления каждого `cointcore.kit.<name>`.

### Донатный ранг (безлимит по количеству, выборочно)

1. Создать набор: `/kit create mykit` (по умолчанию безлимит).
2. Добавить `cointcore.kit.mykit` в нужный ранг ServerUtilities (или использовать `cointcore.kit.all` только для staff).
3. Игроки ранга используют `/kit claim mykit` сколько угодно раз.

### Разовая или ограниченная покупка набора

1. Создать набор с лимитом: `/kit create shopkit 3`.
2. Выдать остаток игроку: `/kit set shopkit PlayerName 3` (или `add`).
3. Игрок забирает наборы без ранговой привилегии, пока не исчерпает лимит.

### Покупка «разовых» получений безлимитного набора (например `fusion`)

1. Набор создан без лимита: `/kit create fusion`.
2. После оплаты: `/kit add fusion PlayerName 10`.
3. Игрок 10 раз забирает набор без ранга; затем нужен донат/`cointcore.kit.fusion` или `cointcore.kit.all`.

## 7) Рекомендации по эксплуатации

- Давайте наборам короткие имена в едином стиле: `starter`, `weekly`, `vip`.
- Перед массовыми изменениями тестируйте два кейса:
  - игрок с ранговым доступом;
  - игрок без ранга, но с персональным остатком использований.
- Для спорных кейсов проверяйте:
  - наличие набора в `/kit list`;
  - заполненность инвентаря;
  - корректность выданных ранговых permission в ServerUtilities;
  - остаток по `set`/`add` при лимитных наборах.
